## 1. Why this file exists

Per Jack (decision 17, `sources/jack-decisions.md`): the way to expand the discrepancy catalog systematically is to **map all the data each system collects, find the common data, and validate that the "common" fields really are the same real-world fact** — then every validated overlap becomes a candidate discrepancy rule. This file is that map. Per decision 16, the goal in this phase is **complete detection** — surface every conflict; the source-of-truth/authority logic comes later.

**Phase-scoped authority — a decided slice of the deferred logic (per Jack, decision 18 (2026-07-12)).** Decision 16 deferred *authority*, but decision 18 now names the primary-source CUSTODIAN by lifecycle segment for the construction→leasing spine: during renovation (PH-3) **Buildertrend is the source and the Podio Renovation Phase is a MIRROR of it**; after the turnover handoff (PH-4+) **AppFolio is primary for day-to-day management** — rent, occupancy, owner (rows 3/4/2) — and **Podio is a compliance supplement only** (row 12: lead certs / rental licenses / MDE, which AppFolio doesn't track). This reconciles with — it does not overturn — decision 16: the segment custodian is DECIDED, per-field conflict-resolution is still deferred, and in this phase the engine still surfaces *every* conflict. The custodian only tells you which side is the stale copy; it does not suppress the finding.

Two rule **classes** fall out of the map:

- **Value-mismatch** (the existing D-rule pattern): two+ systems store the same fact and record it *differently*. (D11 owner, D20 rent are examples.)
- **State-consistency** (Jack's new class, seeded by his own two examples): two systems are each internally valid but describe *states that cannot legally coexist* — e.g. "in Design & Scope while Occupied," or "in Buildertrend while still Waiting to Close." These don't compare a shared value; they compare *lifecycle position* across systems.

Helm already runs **9 value-mismatch rules** (D1, D5, D7, D9, D11, D14, D17, D18, D20 — note the tour only showed 7 firing; D5 and D9 also exist). Everything proposed here is deduped against those 9.

Reading note on proposed IDs: value-mismatch candidates reuse the **unused numbers in the existing D1–D20 range** (D2, D3, D4, D12, D13, D15, D16, D19); state-consistency candidates start a fresh block at **D21+**. The code read confirms these gaps are genuinely free to claim: D2/D3/D4/D6/D8/D10/D12/D13/D15/D16/D19 were **never implemented** — no code, comments, or deleted files in git history (per Helm codebase read 2026-07-11, findings §1). Nothing is reserved for a *retired* rule, so no renumbering is needed on that account. [confirm with the developer: are any of these numbers reserved for a rule you have *planned* but not yet built? If so, renumber.]

## 5. False-positive guards (catalog-wide)

Every rule above must respect these, learned from the existing 9 and the GlobiFlow inventory:

1. **Multifamily fan-out.** Property-level Podio rows vs per-unit AppFolio rows produce spurious mismatches (already hits D20). Compare at `unit_id` where possible; require single-unit or a unit-level match before firing property-vs-unit comparisons. (D16 exists partly to *detect* this.)
2. **Turnover clears fields by design.** A GlobiFlow ("turnover = unset water billable, rent, program, anniversary month") deliberately blanks the Podio rent/program on turnover. Empty Podio rent/program mid-turn is **expected state** — classify, don't flag. Gate rent/program rules on `property_status`≠'Turnover'. (This 'Turnover' status is the **DM→DP vacancy-side turn** — per Jack, decision 19 (2026-07-12) — the managed unit went vacant and DP is making it rent-ready; not the DP→DM 'Handed Off - Rental' completion of decision 18.)
3. **Placeholder owners.** "Dominion Properties", "DP new unit", "3rd Party" are catch-alls, not real LLCs — exclude (D11 already does).
4. **Match-key integrity.** Compare only records already resolved to the same `property_id`; never fuzzy candidate pairs (near-miss addresses are how matching legitimately fails upstream). D13 is the meta-rule for this.
